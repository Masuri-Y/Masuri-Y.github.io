LVM逻辑卷和文件系统

逻辑卷的组成是，集成各种空间组合成物理卷，然后集合物理卷组合成一个卷组，然后在卷组上创建逻辑卷。

所以要创建逻辑卷首先需要创建物理卷，然后组合物理卷创建卷组，最后创建可以使用的逻辑卷。

逻辑卷创建方式

以下为逻辑卷的创建方式

实验环境：

1.一个mbr分区的5G空间(sdb1)
2.一个gpt分区的5G空间(sdc1)
3.一个未分区的磁盘(sdd)

创建实验环境

1.建立一个mbr分区的5G空间(sdb1)

2.建立一个gpt分区的5G空间

3.一个未分区的磁盘

至此实验环境搭建完毕

创建LVM

  • 1.创建物理卷（PV）

将3块磁盘空间全部转换为物理卷（PV）

  • 2.创建卷组（vg）

出于后期增加卷组考虑此处先使用2块磁盘空间组成vg

将mbr分区的sdb1和gpt分区的sdc1组成卷组

  • 3.创建逻辑卷（lv）

1.创建一个6G的逻辑卷,一个3G的逻辑卷

至此2个逻辑卷已经创建完成。接下来需要在逻辑卷上创建文件系统了

创建文件系统

文件系统有很多种类，此处以xfs和ext4为例进行建立

ext4文件系统

1.在/dev/vg0/test1上建立ext4

2.将ext4文件系统挂载至/data/test1下

xfs文件系统

1.在/dev/vg0/tset2上建立xfs文件系统。

2.将文件系统挂载至/data/test2下

2个文件系统已经创建完毕,由于后续实验需求现在两个磁盘上分别创建几个文件

逻辑卷的容量增加和减少

一、分别对逻辑卷test1及test2增加1G容量

1.查看卷组的相关信息

此时卷组中只剩下不足1G容量需要增加物理卷才能满足需求

2.对vg0增加物理卷，将刚才未加入卷组的sdd加入卷组

3.对ext文件系统的逻辑卷test1进行扩容

4.对xfs文件系统的逻辑卷test2进行扩容

5.查看文件系统内的文件是否丢失

对逻辑卷做扩容可以在线操作并且不会造成数据丢失的

二、对逻辑卷做缩减

逻辑卷是否能缩减取决于文件系统是否支持，XFS文件系统不支持缩减，所以此处以ext4为例，将逻辑卷test1缩减至4G，对文件系统做缩减需要先对其内的数据做备份防止数据丢失

1.逻辑卷缩减首先要将设备离线

2.对逻辑卷缩减前首先先要将文件系统的空间缩减

3.对逻辑卷进行缩减

4.挂载文件系统，查看内部数据
