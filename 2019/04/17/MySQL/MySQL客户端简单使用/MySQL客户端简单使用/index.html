MySQL客户端简单使用

发表于2019-04-17|更新于2020-11-21|MySQL
|阅读量:

本节主要说明MySQL的客户端的使用方式

MySQL账号

mysql用户账号由两部分组成

USERNAME：用户名

HOST：限制此用户可以通过哪些远程主机连接MySQL服务器

  • HOST支持使用通配符”%”，”_”
  • %匹配任意长度的任意字符，如192.168.0.0/255.255.0.0 或172.16.%.%
  • _匹配任意长度的单个字符。

MySQL的登录

MySQL在登录时需要使用-u指定账号，-p指定密码

由于MySQL刚安装好，管理员账号是没有密码的，所以不需要密码也可以登录

不光没密码能登陆，连账号都不输入也能登录，MySQL默认使用root登录

查看当前MySQL登录用户

登录MySQL后，使用SELECT USER();可以查看当前MySQL的登录用户是谁。

查看MySQL数据库版本

登录MySQL后，使用SELECT VERSION();可以查看当前MySQL数据库的版本号

查看MySQL客户端使用帮助

登录MySQL后使用\h可以查看MySQL客户端的使用帮助

显示用户的状态信息

在登录后使用STATUS;可以显示当前登录用户的状态信息

查看数据库

登录数据库后，使用SHOW DATABASES;可以查看MySQL中所有的数据库

数据库在磁盘上也是以文件的方式存放的，yum安装的数据库默认存放在/var/lib/mysql目录中，其中information_schema存在于内存中

指定使用哪个数据库

使用USE加上数据库名字，可以指定使用哪一个数据库

查看当前数据库中的表

在选中要使用的数据库后可以使用SHOW TABLES;来查看当前库中的所有表

查看表的字段

查看某表中的所有字段，可以使用DESC命令加上所要查看的表的表名

查看表中指定字段的信息

使用select命令，指定字段的名称和所要查询的表，可以从表中查看所要查看的字段信息

MySQL的命令执行方法

MySQL除了交互式状态下执行命令，还支持非交互式状态下执行命令

使用重定向执行MySQL命令

使用-e选项执行MySQL命令

除了非交互式状态，MySQL还能在交互式状态下使用source调取系统的文件作为命令来使用

MySQL的登陆时默认数据库

MySQL在登陆的时候使用-D 加上数据库名，还可以指定默认的数据库

MySQL的客户端配置文件

MySQL的客户端配置文件在/etc/my.cnf.d/mysql-clients.cnf

MySQL客户端更改提示符

MySQL客户端也和linux的客户端一样可以对提示符进行修改，需要在MySQL中使用prompt命令加上相关的参数进行修改

在命令行所作的配置只是零时有效，若要长期有效还是需要写入配置文件中

在登陆时使用–print-defaults查看客户端登陆时默认所带的选项

MySQL的服务器端配置文件

MySQL的服务器端配置文件为类ini格式配置文件
